# 图片 dHash 动态分桶与严格两两分组修改计划

> 日期：2026-07-16  
> 状态：计划已生成，等待执行  
> 范围：只调整图片 dHash 候选召回、严格分组、配置与相关测试；不修改图片 dHash 生成算法和视频相似规则。

## 1. 目标

1. 图片相似阈值使用配置项 `n`，判定条件为：

```text
HammingDistance(left, right) <= n
```

2. 将 64 位图片 dHash 等分为 `n+1` 个连续、不重叠且非空的段，按段序号写入对应桶。
3. 仅当两个图片至少有一个相同序号的 dHash 段完全相等时，才进入真实汉明距离计算。
4. 同一相似组内任意两个不同图片内容的汉明距离都必须 `<= n`。
5. 禁止使用基准 dHash、代表 dHash 或中心文件作为分组判定依据。
6. 通过分桶、去重、快速拒绝和组内早停降低实际比较次数。
7. `n` 在配置页面中可设置，配置、任务快照和索引命名空间保持一致。

## 2. 已确认规则

### 2.1 n 的含义

`n` 表示图片相似允许的最大汉明距离，包含边界：

```text
distance <= n
```

配置范围：

```text
0 <= n <= 15
```

默认值：

```text
n = 4
```

配置非法时禁止保存或启动任务，不进行静默截断。

### 2.2 分桶正确性

把 64 位 dHash 切成 `n+1` 段。若两个 dHash 的汉明距离不超过 `n`，最多有 `n` 个段包含差异位，因此至少有一个段完全相等。

所以：

```text
HammingDistance(left, right) <= n
    =>
至少一个相同序号的分段完全相等
```

分桶只负责无遗漏召回候选，不能直接证明两个图片相似；所有候选仍需执行完整 64 位汉明距离复核。

## 3. 分段规则

### 3.1 均匀切分

按 dHash 从高位到低位固定切分：

```text
segment_count = n + 1
base_width = 64 / segment_count
remainder = 64 % segment_count
```

前 `remainder` 段宽度为：

```text
base_width + 1
```

其余段宽度为：

```text
base_width
```

例如 `n=4`：

```text
64 位 -> 13、13、13、13、12
```

必须保证：

- 所有段总位数等于 64。
- 段之间没有重叠。
- 没有遗漏位。
- 没有空段。
- 相同 n 下分段布局始终一致。

### 3.2 桶键

图片桶键必须至少包含：

```text
图片桶格式版本
媒体算法版本
n
分段序号
分段位宽
分段值
完整图片 dHash
```

完整图片 dHash 作为键后缀，用于压缩相同视觉签名，避免热门桶形成大量重复索引记录。

不同 n 必须使用完全隔离的 RocksDB 键空间，禁止用新 n 查询旧 n 的桶。

## 4. 候选索引流程

### 4.1 写入

对每个有效图片内容：

1. 检查媒体类型必须为图片。
2. 检查 dHash 存在且不是默认无效值。
3. 按当前任务快照中的 n 切成 `n+1` 段。
4. 每个段按自身序号写入对应桶。
5. 相同完整 dHash 的索引记录允许覆盖，只保留一个视觉签名候选。

单个图片写入索引数量为：

```text
n + 1
```

### 4.2 查询

对待查询图片：

1. 使用与索引一致的 n 和分段规则生成 `n+1` 个桶前缀。
2. 依次查询相同段序号、位宽和段值的桶。
3. 合并所有桶返回的候选。
4. 按候选 SHA-512 或完整视觉签名去重。
5. 排除查询内容自身。
6. 只把去重后的候选交给真实汉明距离计算。

一个候选即使命中多个段，也只能进入一次后续计算。

## 5. 严格两两分组

### 5.1 分组不变量

最终每个图片组必须满足：

```text
对于组内任意两个不同 SHA-512 内容 a、b：
HammingDistance(a.dhash, b.dhash) <= n
```

不能因为以下情况把内容合并到同组：

```text
A 与 B 相似
B 与 C 相似
A 与 C 不相似
```

上述数据必须被拆成至少两个组，禁止使用并查集连通分量产生链式传递合并。

### 5.2 禁止基准 dHash 分组

不得采用以下规则：

```text
成员只需要与组代表相似
成员只需要与第一个文件相似
成员只需要与中心 dHash 相似
```

组根、首个 SHA-512 或组 ID 可以作为存储键和稳定标识，但不能作为相似性基准。

### 5.3 新成员加入组

对一个新图片内容：

1. 通过分桶召回与其直接相似的候选内容。
2. 根据候选内容定位可能接纳该内容的候选组。
3. 对每个候选组，逐一读取组内所有不同 SHA-512 内容的图片 dHash。
4. 新内容必须与组内每个内容都满足 `distance <= n`。
5. 任意一项距离超过 n，立即拒绝该候选组。
6. 只有全部成员校验通过，才允许加入。
7. 没有任何候选组通过时，新建组。

相同 SHA-512 的多条文件路径不重复参与 dHash 比较，在内容分组完成后统一展开到报告成员。

### 5.4 多个组都可接纳时

为保证结果稳定，内容按 SHA-512 字典序处理。多个候选组都满足完整链接时，按以下顺序选择：

1. 新内容到组内成员的最大汉明距离更小。
2. 最大距离相同时，平均汉明距离更小。
3. 仍相同时，选择稳定组标识字典序更小的组。

上述选择只用于多个合法组之间的确定性归属，不能代替组内完整两两校验。

## 6. 剪枝优化

按以下顺序执行：

1. **无效数据剪枝**：非图片、缺失 dHash、默认无效值直接跳过。
2. **桶剪枝**：没有任何相同分段桶的内容不进入真实距离计算。
3. **候选去重**：同一候选命中多个桶时只保留一次。
4. **自身剪枝**：相同 SHA-512 内容不与自身比较。
5. **路径折叠**：同 SHA-512 的多个路径只进行一次内容级比较。
6. **媒体类型剪枝**：图片候选组不读取视频成员。
7. **真实距离快速判等**：完整 dHash 相等时距离直接为 0。
8. **完整链接早停**：发现任意组成员距离超过 n 后，立即停止检查当前候选组。
9. **候选组剪枝**：只有包含至少一个直接候选成员的组才进入完整链接校验。
10. **取消剪枝**：桶遍历、候选遍历、候选组遍历和组内成员遍历均响应取消请求。

64 位 dHash 的单次异或加硬件 popcount 成本很低，不能为了“逐段早停”引入比完整 `popcount(left ^ right)` 更大的循环开销。执行阶段应以基准测试决定是否保留逐段累计距离；分桶和完整链接早停是必须保留的主要剪枝。

## 7. 配置与生命周期

配置分区：

```json
{
  "dhash_similarity": {
    "image_max_hamming_distance": 4
  }
}
```

配置页面增加：

```text
图片最大汉明距离 n
```

提示内容：

```text
有效范围 0～15。
候选索引分为 n+1 段。
n 越大，候选数量和完整链接校验成本通常越高。
```

生效规则：

- 保存配置时执行范围校验。
- 报告启动时冻结 n，运行中修改配置不影响当前任务。
- 扫描任务快照和断点保存 n。
- 旧配置或旧快照缺少字段时迁移为默认值 4。
- n 变化后必须重建对应的图片候选桶，不能混用旧桶。

## 10. 测试计划

### 10.1 分段测试

对 `n=0～15` 分别验证：

- 段数等于 `n+1`。
- 总位数等于 64。
- 不存在空段。
- 分段布局稳定。
- 任意距离 `<= n` 的样本至少共享一个相同序号的完整段。

### 10.2 候选测试

- 同一候选命中多个桶时只返回一次。
- 不同 n 的索引互不召回。
- 距离 `<= n` 的候选不漏召回。
- 桶命中但距离 `> n` 的候选被真实距离复核拒绝。
- 相同完整 dHash 的热门内容不会形成重复回调。

### 10.3 分组测试

链式数据：

```text
distance(A, B) <= n
distance(B, C) <= n
distance(A, C) > n
```

验收结果：

```text
A、B、C 不得全部进入同一组
```

此外必须遍历每个生成的图片组，对任意两个不同 SHA-512 内容重新计算距离，确保全部 `<= n`。

### 10.4 配置测试

- 默认 n 为 4。
- n=0 和 n=15 可以保存并启动。
- n>15 被拒绝。
- 旧配置迁移后 n 为 4。
- 旧任务快照恢复后 n 为 4。
- 当前任务不受运行中配置修改影响。

### 10.5 性能与取消测试

- 大桶场景不会重复比较同一候选。
- 候选组出现首个超限成员后立即停止。
- 取消报告时能从桶遍历和组内两两校验中及时退出。
- 对实际 MySQL 全量数据记录修改前后的候选数、真实比较次数和报告耗时。

## 11. 验收标准

1. 配置页面可设置 n，范围 `0～15`，默认 `4`。
2. 图片索引按 n 动态生成 `n+1` 个单段桶。
3. 所有距离 `<= n` 的图片对都不会因分桶而漏召回。
4. 桶命中后必须执行真实 64 位汉明距离计算。
5. 同组任意两个不同图片内容的汉明距离都 `<= n`。
6. 分组代码不使用基准 dHash、代表 dHash 或首个成员替代完整两两校验。
7. 不存在相似关系的链式传递合并。
8. 同 SHA-512 多路径不重复参与计算，但报告中仍完整展示。
9. 不同 n 的 RocksDB 索引完全隔离。
10. 取消操作能及时终止长桶和组内校验。
11. Debug/Release x64 构建成功，相关自动测试全部通过。
