# 删除 SHA-512 与 dHash 重复报告修改计划

> 日期：2026-07-16  
> 状态：已完成修改；Debug/Release x64 构建及测试通过

## 1. 已确认需求

1. 在“重复报告”窗口增加两个独立按钮：
   - `删除 SHA-512 重复报告`
   - `删除 dHash 重复报告`
2. 不增加“报告生成路径”配置。
3. 删除操作只清理对应类型的重复报告数据，不删除：
   - 媒体文件。
   - 文件路径及扫描记录。
   - 已计算的 SHA-512。
   - 图片 dHash、视频六帧 dHash及其他媒体特征。
   - MySQL 中的任何数据。
4. 两类报告互不影响。删除 SHA-512 报告时保留 dHash 报告，删除 dHash 报告时保留 SHA-512 报告。

## 2. 当前实现盘点

### 2.1 报告存储位置

当前两类报告均存放在主 RocksDB 的 `ExactIndex` Column Family 中：

| 报告类型 | 活动报告键 | 报告数据前缀 |
| --- | --- | --- |
| SHA-512 精确重复报告 | `report/exact/active` | `report/exact/<generation>/` |
| dHash 内容相似报告 | `report/similar/active` | `report/similar/<generation>/` |

dHash 报告生成期间还会使用：

```text
report/similar/work/<generation>/
```

正常发布新报告时，`DuplicateReportStore::Publish()` 会删除上一代已发布报告。因此正常情况下每种类型只有一份活动报告；但程序异常退出后仍可能遗留未发布代或工作区数据。

### 2.2 当前 GUI 状态

“重复报告”窗口目前具有：

- 从 MySQL 生成 SHA-512 精确报告。
- 手动计算 dHash 相似报告。
- 加载已发布报告。
- 取消正在进行的报告生成。
- 根据报告选择并永久删除实际媒体文件。

目前没有“仅删除报告数据”的入口。“删除选中文件”属于实际媒体文件永久删除，不能与本次报告清理按钮共用确认弹窗或业务状态。

## 3. 删除语义

### 3.1 删除范围

核心层为 `DuplicateReportStore` 增加按报告类型清理的方法。该方法删除指定类型的完整报告命名空间：

```text
SHA-512：report/exact/
dHash：  report/similar/
```

这样可以同时清理：

- 当前活动报告指针。
- 当前活动报告的组记录、摘要和数量。
- 因异常退出遗留的旧代或未发布代数据。
- dHash 报告异常退出后遗留在正式报告命名空间下的工作数据。

不清理其他 RocksDB 前缀，也不访问 MySQL。

### 3.2 安全顺序

删除时按照以下顺序处理：

1. 先同步删除对应类型的 `active` 键，使 GUI 和后续读取不再发现该报告。
2. 再使用现有 `RocksStore::DeletePrefix()` 按固定批量删除该类型报告前缀下的剩余数据，避免一次性把全部键加载到内存。
3. 即使当前没有活动报告，也继续执行前缀清理，以清除异常退出遗留的数据，并保证重复点击是幂等操作。
4. 任一步失败时返回明确错误，不把失败显示为成功；用户可再次执行清理。

报告生成、扫描、实际媒体文件永久删除或另一项报告清理正在运行时，不允许启动新的报告清理。

## 4. GUI 设计

### 4.1 按钮位置

在“重复报告”窗口现有“加载已发布报告”操作区附近增加两个危险操作按钮：

```text
[删除 SHA-512 重复报告] [删除 dHash 重复报告]
```

两个按钮始终明确对应固定报告类型，不依赖当前选中的页签，避免用户切换页签后误删另一类报告。

### 4.2 二次确认

两个按钮分别打开对应的确认弹窗。弹窗必须明确显示：

- 即将删除的报告类型。
- 删除后需要重新生成报告才能再次查看重复组。
- 不会删除媒体文件、SHA-512、dHash、扫描记录或 MySQL 数据。

确认按钮使用危险操作样式，取消按钮不产生任何状态变化。

### 4.3 后台执行与状态

报告数据可能包含大量组记录，不能在 ImGui 帧线程中直接执行前缀删除。GUI 增加独立的后台报告清理任务：

1. 用户确认后记录待删除的报告类型。
2. 在后台线程调用核心层清理接口。
3. 清理期间禁用报告生成、加载、两种报告删除和实际媒体文件永久删除入口。
4. 界面显示：
   - `正在删除 SHA-512 重复报告……`
   - 或 `正在删除 dHash 重复报告……`
5. 应用退出或关闭 RocksDB 前，等待清理线程安全结束。
6. RocksDB 返回错误或线程抛出异常时，沿用现有完整异常处理和执行日志机制记录错误。

本次不增加无法准确计算的百分比进度条，仅显示正在处理状态，避免为 RocksDB 分批删除伪造进度。

### 4.4 删除成功后的界面清理

删除成功后：

1. 如果当前窗口正在显示被删除的报告类型：
   - 清空活动报告代号和组数。
   - 清空摘要、虚拟列表行索引、组缓存和删除选择。
   - 关闭该报告打开的重复组详情窗口。
   - 清理报告详情使用的临时缩略图和纹理。
   - 将滚动位置恢复到顶部。
2. 如果当前显示的是另一类报告，则保留另一类报告的列表和详情状态。
3. 显示明确成功消息：
   - `SHA-512 重复报告已删除。`
   - `dHash 重复报告已删除。`
4. 当前本来就没有对应报告时，仍视为幂等成功，并提示不存在可显示的该类报告。

## 7. 验收标准

1. 重复报告窗口显示两个独立按钮：
   - `删除 SHA-512 重复报告`
   - `删除 dHash 重复报告`
2. 点击任一按钮不会立即删除，必须经过对应类型的二次确认。
3. 删除 SHA-512 报告后：
   - SHA-512 页签显示尚未生成报告。
   - 原 SHA-512 报告组、摘要和活动键均不存在。
   - dHash 报告仍可正常加载和查看。
4. 删除 dHash 报告后：
   - dHash 页签显示尚未生成报告。
   - 原 dHash 报告组、摘要、活动键和遗留工作数据均不存在。
   - SHA-512 报告仍可正常加载和查看。
5. 删除报告不会删除或修改任何媒体文件、原始 SHA-512、dHash、扫描记录和 MySQL 数据。
6. 大型报告清理期间 GUI 保持响应，不在 ImGui 帧线程执行批量 RocksDB 删除。
7. 扫描、报告生成、实际文件删除或报告清理进行中时，两个报告删除按钮不可用。
8. 删除当前可见报告后，列表、详情窗口、选择状态和临时预览资源被正确清空。
9. 删除非当前可见报告时，当前另一类报告的界面状态不受影响。
10. 对不存在的报告重复执行删除不会报错或崩溃。
11. 清理失败时界面显示具体错误，执行日志中存在对应失败记录。
12. 不出现“报告生成路径”配置项，现有配置文件格式不变。
13. Debug/Release x64 构建成功，现有测试和新增报告删除测试全部通过。
