# 重复结果纯文本 UI 改造计划

> 日期：2026-07-15  
> 状态：已按推荐交互口径执行完成

## 1. 目标

将当前“重复报告”中的折叠式缩略图表格改成轻量的纯文本结果列表：

- 一组重复或相似媒体直接连续显示，不再使用展开、折叠交互。
- 主窗口不显示缩略图，只显示组摘要和组内文件文本。
- 被选中的项目使用整行高亮底色，不再依赖复选框作为主要视觉提示。
- 点击组入口后，在另一个“重复组详情”窗口显示缩略图、质量参数和相似度证据。
- 保留现有分页、报告类型切换、保留策略、条件选择和永久删除安全约束。

## 2. 当前实现

当前 `VideoScApp::RenderDuplicateReportWindow` 对每个 `DuplicateGroup` 使用 `ImGui::CollapsingHeader`：

1. 组未展开时只显示组标题。
2. 展开后创建六列表格并加载每个成员的缩略图。
3. 文件删除选择使用复选框。
4. 图片和视频纹理都在报告主窗口中按可见区域加载。

现有主 DockSpace 已经把“重复报告”作为独立 ImGui 窗口，因此新增“重复组详情”可继续使用同一应用内的 Dockable ImGui 窗口，无需创建第二个 Win32 进程或第二套 D3D11 渲染循环。

## 3. 推荐交互设计

### 3.1 重复报告主窗口

每个重复组固定显示为一个连续文本块，不提供折叠按钮：

```text
重复组 1024 · 4 个文件 · 可释放 1.82 GiB · 点击查看详情
  [保留] E:\media\a.mp4 · 1920×1080 · 624 MiB · PhysicalDrive1
  [删除] E:\backup\a.mp4 · 1920×1080 · 624 MiB · PhysicalDrive3
  [删除] E:\temp\a.mp4 · 1920×1080 · 624 MiB · PhysicalDrive3
  [保留] E:\archive\a.mp4 · 1280×720 · 410 MiB · PhysicalDrive4
```

- 组标题使用一个纯文本 `Selectable`，点击后选中该组并打开详情窗口。
- 成员使用无边框的纯文本 `Selectable`，整行底色表达当前选择状态。
- 文本行至少包含：保留/删除状态、完整路径、分辨率、大小和磁盘。
- 路径过长时在主列表中省略显示，鼠标悬停显示完整路径；详情窗口始终显示完整路径。
- 组与组之间使用留白或细分隔线区分，不使用卡片边框和折叠箭头。
- 相似报告的组标题增加平均汉明距离摘要；精确报告不显示无意义的相似度字段。

### 3.2 高亮规则

- 待删除成员：使用当前主题的选中底色，文字保持正常可读。
- 当前打开详情的组：组标题使用更明显的强调底色。
- 保留成员：默认透明底色，并显示 `[保留]` 文本标记。
- 禁止通过高亮选择把同一组所有成员都设为删除；继续复用现有“至少保留一个副本”校验。
- 条件选择当前页或全部报告后，当前页成员行立即同步高亮。

### 3.3 重复组详情窗口

新增 `RenderDuplicateReportDetailWindow`，只在已经选择组时显示：

- 窗口标题：`重复组详情 · 组 {group_id}`。
- 顶部显示组类型、成员数量、可释放空间、算法版本和相似度摘要。
- 中部显示现有缩略图或视频 2×3 拼图，以及完整路径、磁盘、大小、分辨率、码率和修改时间。
- 相似组额外显示成员之间的逐帧汉明距离、平均距离和时长差。
- 详情窗口中的成员选择状态与主窗口共用 `selected_for_deletion`，不得复制一份独立状态。
- 关闭详情、切换报告类型、换页或重新加载报告时，清理失效的组选择和纹理。

## 4. 状态设计

在 `VideoScApp` 中增加最小 UI 状态：

- `std::optional<std::uint64_t> m_selectedReportGroupId`：当前高亮并显示详情的组 ID。
- `bool m_showReportDetailWindow`：详情窗口是否打开。

不保存 `DuplicateGroup*` 或成员引用。每帧根据 `group_id` 从 `m_reportPageGroups` 查找，避免换页和重新加载后悬空引用。

删除选择仍只使用现有 `DuplicateGroup::selected_for_deletion` 和 `retained_path_id`。报告模型、RocksDB 格式和删除执行器均不改动。

2. 增加当前详情组 ID 和窗口显示状态。
3. 如成员选择逻辑需要在两个窗口复用，提取一个仅作用于当前组的私有方法；不新增公共接口。

### `VideoScGUI/VideoScApp.cpp`

1. `Render()` 中增加详情窗口渲染调用。
2. 重写 `RenderDuplicateReportWindow` 的组列表区域：
   - 删除 `CollapsingHeader`。
   - 删除主列表中的缩略图表格。
   - 使用纯文本 `Selectable` 绘制组标题和成员行。
   - 点击组标题时更新详情状态。
3. 新增 `RenderDuplicateReportDetailWindow`，迁移并复用现有缩略图、媒体参数和相似度证据展示。
4. 换页、切换报告、重新加载报告时清理失效详情选择。
5. 调整纹理回收时机，确保只有详情窗口打开时加载报告纹理。

### 不修改范围

- `DedupCore` 报告生成和持久化格式。
- SHA-512、dHash 和相似组算法。
- 分页读取逻辑。
- 保留策略和永久删除复核流程。
- D3D11 渲染后端和 Win32 主窗口生命周期。

## 7. 验收标准

1. 主窗口不再出现组展开箭头、折叠动画或缩略图表格。
2. 当前页所有组及其成员都以纯文本连续显示。
3. 被选中的成员整行高亮，条件选择后高亮立即刷新。
4. 同组最后一个保留成员不能被选为删除。
5. 点击组标题能打开或切换到对应详情窗口。
6. 详情窗口显示该组全部成员、预览和完整对比信息。
7. 换页或切换报告类型后不会显示上一页的悬空详情。
8. 详情窗口关闭时，主列表滚动不会继续加载缩略图纹理。
9. 现有分页、报告生成、删除确认和删除结果提示保持可用。

## 8. 已采用的交互口径

1. “选中高亮”解释为“成员被选为待删除时高亮”；当前打开详情的组标题使用另一档强调色。
2. “另一个窗口”使用同一主程序内可停靠的 ImGui 窗口，不新增 Win32 原生窗口和第二套渲染循环。
