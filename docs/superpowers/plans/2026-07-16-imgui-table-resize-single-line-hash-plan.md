# ImGui 表格列宽、单行文本与媒体哈希显示修改计划

> 日期：2026-07-16  
> 状态：已完成修改；Debug/Release 构建及测试通过

## 1. 修改目标

1. 在重复组详情界面新增 dHash 显示；图片、视频成员均显示完整 SHA-512 与对应 dHash。
2. 项目内所有 `ImGui::BeginTable()` 表格都允许用户用鼠标拖动列边界调整宽度。
3. 表格内长文本不自动换行，保持单行高度；超出列宽的内容由列裁剪。
4. 单行内容被裁剪时，鼠标悬停显示完整文本，避免信息丢失。

## 2. 实现方案

1. 在 `VideoScApp` 中增加私有静态表格文本方法，统一负责单行绘制与悬停提示，不创建跨模块公共工具。
2. 扫描路径进度、磁盘进度、相似度证据、重复组详情、截图结果、文件检索结果六类表格统一增加 `ImGuiTableFlags_Resizable`。
3. 移除表格单元格内的 `ImGui::TextWrapped()`，路径、磁盘、成员名称、说明和时间文本改用单行方法。
4. 重复组详情保留现有 SHA-512 显示，并新增可复制的 dHash 单行字段：
   - 图片：显示 128 位小写 SHA-512；新增 `0x` 加 16 位图片 dHash。
   - 视频：显示 128 位小写 SHA-512；新增六个 `0x` 加 16 位逐帧 dHash，并标明对应帧序号。
   - dHash 不存在或无效时显示明确的无效状态，不以默认值冒充有效结果。
5. 不改变表格外弹窗、配置提示等文本的自动换行行为。

## 3. 验证

1. Debug/Release x64 构建通过。
2. `DedupTests` 全部通过。
3. 静态检查所有 `BeginTable` 均包含 `Resizable`，表格范围内不再使用 `TextWrapped`。
4. 检查图片详情存在一个 dHash 字段，视频详情存在六个带帧序号的 dHash 字段，且内容可完整查看和复制。

## 4. 执行结果

1. 六张 ImGui 表格均已启用 `ImGuiTableFlags_Resizable`。
2. 表格内长文本改为单行裁剪，鼠标悬停可查看完整内容；表格内不再调用 `TextWrapped()`。
3. 重复组详情明确显示 SHA-512、图片 dHash 和六帧视频 dHash：
   - 字段标签与值同行显示，值保持只读可复制。
   - 缺失值显示“无（未计算）”。
   - 图片 dHash 为 0、视频任意帧 dHash 为 0 时，在对应值后显示“无效”。
4. Debug/Release x64 均构建成功，两个配置的 `DedupTests` 均为 33/33 通过。
