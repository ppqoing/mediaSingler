# 01－图片与视频严格汉明距离分组修改计划

> 日期：2026-07-17  
> 状态：已执行，等待最终验收
> 范围：只修改 dHash 相似报告分组，不改变 SHA-512 精确报告

## 1. 修改目标

相似报告中的每个主组必须满足：

```text
图片：组内任意两个不同视觉签名的汉明距离 <= 图片配置阈值
视频：组内任意两个不同视觉签名的 6 帧平均汉明距离 <= 视频配置阈值
```

禁止只使用组根、首成员、平均 dHash、中心 dHash或相似关系传递闭包决定整组合并。

## 2. 当前实现

- `DuplicateReportGenerator::GenerateSimilar()` 已通过候选边和 `group-of` 执行 complete-link 分组。
- 图片已经使用配置阈值；视频仍使用固定平均距离规则。
- 分组按稳定 SHA-512 字典序提交，结果具备确定性。
- 相同视觉签名的多个 SHA-512 内容会压缩到同一代表。

本次复用现有流程，不重新设计报告存储体系。

## 3. 修改方案

1. 将图片和视频的最终比较统一收口到 `CompareVisuals()`，显式传入冻结后的媒体规则。
2. 候选召回只决定需要比较的签名对；只有真实比较通过后才写入规范化相似边。
3. 当前签名加入候选组前，逐一检查它与组内全部不同视觉签名之间是否存在已验证相似边。
4. 任意一项不存在相似边，立即拒绝该候选组并剪枝，不再检查剩余成员。
5. 多个组均可接纳时按以下顺序确定唯一归属：
   - 到组内成员的最大距离更小。
   - 最大距离相同时，平均距离更小。
   - 仍相同时，组根字典序更小。
6. `group-of` 提交保持单线程；纯距离计算可由 dHash 报告线程池并行完成。
7. 为每个组记录不同签名数与已检查关系数；发布前验证 `k` 个签名已经完成 `k*(k-1)/2` 个直接关系检查。
8. 图片跨严格组直接相似关系继续只读显示，不参与可释放大小和删除选择。
9. 本条不新增视频跨组关系界面。

## 4. 数据边界

```text
文件路径 -> SHA-512 内容 -> 视觉签名 -> 唯一严格主组
```

- 相同签名内的成员距离视为 0。
- 严格条件在不同视觉签名层验证，避免热门相同 dHash 文件形成路径级平方计算。
- 主组成员仍保存活动文件路径，删除逻辑不直接消费签名边。

## 6. 异常与取消

- 任一候选读取、边写入或组状态读取失败时停止当前 generation。
- 取消后不再提交新组，等待在途纯计算结束，不发布半成品。
- 上一份 active 报告保持不变。
- 失败日志记录当前签名、候选组根、媒体类型、阈值和失败操作。

## 7. 测试与验收

1. 构造 `A-B` 合格、`B-C` 合格、`A-C` 不合格，三者不得进入同组。
2. 枚举每个发布图片组的全部签名对，距离均不超过图片阈值。
3. 枚举每个发布视频组的全部签名对，6 帧平均距离均不超过视频阈值。
4. 线程池大小为 1、2、4 时，组 ID、成员、证据和跨组图片关系完全一致。
5. 单组数百个同签名文件不会产生文件路径级两两边。
6. Debug/Release x64 构建成功，相关自动化测试全部通过。
