# 磁盘拓扑与并发调度模块计划

## 1. 模块目标

在约 10 块 HDD/SSD 混合磁盘上同时读取和计算所有文件的 SHA-512。独立物理盘并行，同一物理盘上的分区共享读取预算。GUI 可为每个物理盘设置读取线程数，并设置全局计算线程数。

## 2. 当前能力与限制

`DiskInfo::GetPhysicalDiskNumber` 已能用 `IOCTL_STORAGE_GET_DEVICE_NUMBER` 查询普通盘符的物理盘编号，但只返回整数，不支持卷挂载目录、介质类型、缓存、网络或虚拟存储语义。当前项目也没有每盘队列和全局计算预算。

## 3. 存储拓扑

### 3.1 `StorageTargetKey`

一次运行中共享同一 I/O 瓶颈的调度键：

- 普通本地卷：设备类型 + 设备编号。
- 无法解析的本地卷：卷 GUID 退化键。
- 网络路径：服务器/共享键，并使用保守并发。
- 虚拟或未知设备：明确类型，不伪装成独立物理盘。

同一物理盘的多个分区必须得到相同 `StorageTargetKey`。

### 3.2 `StorageTargetDescriptor`

包含显示名称、关联卷、介质类型、寻道提示、在线状态、默认读取线程数、用户覆盖线程数、队列容量、缓冲区预算和实时吞吐。

### 3.3 拓扑查询

1. `GetVolumePathNameW` 获取卷挂载点。
2. `GetVolumeNameForVolumeMountPointW` 获取卷 GUID。
3. `IOCTL_STORAGE_GET_DEVICE_NUMBER` 获取设备编号。
4. `IOCTL_STORAGE_QUERY_PROPERTY` 获取寻道惩罚等介质提示。
5. 结果按卷缓存，避免每个文件重复打开卷设备。

保留旧 `GetPhysicalDiskNumber` C ABI，新增类型化拓扑接口供 `DedupCore` 使用。

### 3.4 NTFS 文件物理区间

针对 HDD，调度前尽量取得文件在磁盘上的近似物理起始位置：

1. `FSCTL_GET_NTFS_VOLUME_DATA` 获取每簇字节数。
2. `FSCTL_GET_RETRIEVAL_POINTERS` 获取文件首个数据 extent 的 LCN。
3. `IOCTL_VOLUME_GET_VOLUME_DISK_EXTENTS` 获取卷在物理磁盘上的起始区间。
4. `physical_start = volume_extent_start + LCN * bytes_per_cluster`。

碎片文件以首 extent 作为排序提示，文件本身仍连续读完。稀疏、驻留、加密、非 NTFS、复杂跨盘卷或查询失败时不阻塞扫描，退化为稳定路径顺序。该物理位置只用于本次调度，不作为永久文件身份。

## 4. 每物理盘读取通道

每个 `StorageTargetKey` 创建一个 `DiskChannel`：

- 有界文件工作队列。
- 可配置数量的读取工作线程。
- 固定容量 SHA-512 缓冲区池。
- 暂停、取消、离线和错误状态。
- 当前路径、已读字节、吞吐和完成数量。
- RocksDB 任务领取与完成检查点。

队列工作项只保存记录 ID、路径引用、预期文件身份、大小和修改时间，不复制媒体数据。

## 5. 读取线程配置

### 5.1 默认值

| 介质 | 默认读取线程 | 原因 |
| --- | ---: | --- |
| HDD | 1 | 避免并发随机寻道降低吞吐 |
| SATA SSD | 2 | 提供有限队列深度 |
| NVMe SSD | 2 | 初始保守，允许用户提高 |
| Unknown | 1 | 无拓扑证据时保守运行 |
| Network | 1 | 避免远端突发压力 |

### 5.2 GUI 配置

- 配置页面按物理盘列出名称、卷、介质和线程数。
- 用户可分别修改每个物理盘的读取线程数。
- 配置页面同时提供全局计算线程数和 FFmpeg 单任务内部线程数，避免只控制磁盘线程却让解码线程过度订阅。
- 配置页面提供读取块大小和每盘队列容量，并实时估算全部读取缓冲占用。
- 配置在任务开始时冻结，修改从下一次扫描生效。
- 非法值在开始前拒绝；具体最大值由测试基准确定。
- 设备新增或编号变化时显示为新目标，不能静默套用错误配置。

## 6. 全局计算线程

全局计算线程数独立于读取线程数，控制：

- SHA-512 摘要状态处理。
- 图片解码与 dHash。
- 视频 seek、解码、六帧 dHash。
- 2×3 拼图缩放和编码。
- 内容相似索引与距离计算。

读取线程可在本线程增量喂入 SHA-512，但提交摘要、媒体解码和拼图必须取得计算令牌。全局计算线程数与 FFmpeg 单任务内部线程数都由配置页面设置，防止“物理盘线程 × 解码线程”过度订阅。

## 7. 所有文件 SHA-512 调度

旧方案中的“大小唯一跳过、快速采样后再完整哈希”全部取消。当前规则是：

1. 新文件或变化文件都进入完整 SHA-512 队列。
2. 未变化且 RocksDB 已有有效摘要的文件可增量复用。
3. 同一硬链接内容实体只读取一次，路径共享结果。
4. 一个文件在同一物理盘工作者内连续读取，不把块拆给多个磁盘线程。
5. 文件读取前后元数据变化时结果作废并重试。

HDD 通道优先使用物理区间有界电梯排序；无法取得区间时按稳定路径顺序。SSD 通道不查询 LCN，允许有限并发。

### 7.1 HDD 有界电梯调度

- 从 RocksDB 为当前 HDD 取一个有界窗口，不全量载入千万级文件。
- 在窗口内按 `physical_start` 排序，沿当前方向依次处理，减少磁头来回移动。
- 窗口完成后从下一个键集游标继续；方向到达当前数据端点后反向。
- 排序窗口大小可配置并通过 HDD 基准确定默认值。
- “物理区间优化”开关和排序窗口大小位于配置页面；SSD 不显示可启用的 LCN 排序选项。
- 同一文件领取后从头到尾连续读取，不跨工作线程拆块。
- 同一物理盘配置多个读取线程时仍按区间分配不重叠工作窗口；GUI 对 HDD 并发大于 1 显示寻道风险提示。

### 7.2 坏块与读取超时

- 使用 overlapped 块读取并监测最后一次成功进展时间。
- `ERROR_CRC`、`ERROR_IO_DEVICE`、`ERROR_SECTOR_NOT_FOUND` 等错误先按原块重试一次，再缩小为 64 KiB 重试一次；次数和小块尺寸可配置。
- 默认连续 60 秒没有成功读入字节时调用 `CancelIoEx`，关闭当前文件句柄并标记 `ReadTimeout`。
- 原块重试次数、小块重试次数、小块尺寸和无进展超时均在配置页面设置，控件显示次数、KiB 和秒单位。
- 任一字节无法读取时不提交部分 SHA-512，文件标记 `Unreadable`。
- 错误日志记录物理盘、路径、偏移、长度、系统错误、重试和超时。
- 跳过当前文件后继续同盘下一文件；不设置连续错误熔断，不自动暂停整个物理盘。
- 不使用 `TerminateThread` 强杀未完成内核 I/O；若驱动未立即响应取消，该工作者显示 `CancelPending`，其他工作者和其他磁盘继续。

## 8. 跨盘公平与资源预算

### 8.1 公平调度

- 所有在线物理盘通道可同时运行。
- 计算令牌按轮转或加权公平分配。
- 一个大盘不能占满全部计算线程和 RocksDB 写入能力。
- 一个物理盘离线不停止其他通道。

### 8.2 内存预算

总内存由以下有界部分构成：

`发现批次 + 每盘文件队列 + 每读取线程缓冲区 + 计算工作区 + RocksDB 缓存 + 当前 GUI 页缩略图`

- 读取块大小、每盘队列容量和缩略图缓存预算来自当前任务的只读配置快照。
- SHA-512 缓冲区建议从 1 至 8 MiB 基准选择。
- 线程数增加前先从全局内存预算申请缓冲区。
- 视频六帧处理后立即释放独立帧，只保留 dHash 和最终拼图编码缓冲。
- MySQL 待同步数据写 RocksDB，不留无界内存队列。

### 8.3 RocksDB 写入

所有通道把小型结果写入一个受控批处理层，通过 `WriteBatch` 原子更新任务状态、摘要和待同步项。工作线程不直接等待 MySQL 网络写入。

## 9. 暂停、取消和恢复

- 每个块读取、任务领取和 FFmpeg 采样点之间检查控制状态。
- 暂停时不领取新文件，当前文件到安全边界后等待。
- 取消时未开始任务保留为可恢复状态。
- 关闭顺序：停止发现 → 关闭入队 → 收敛读取线程 → 提交 RocksDB 批次 → join。
- 崩溃后从 RocksDB 未完成工作键重建通道，不依赖退出前内存。

## 11. 实施任务

### M1：拓扑 API

- 卷、设备、介质和错误类型。
- 缓存与旧 C ABI 兼容。

### M2：有界队列和预算

- 队列关闭、取消、唤醒和资源预算。
- 多通道压力测试。

### M3：物理盘配置

- HDD/SSD 默认值。
- GUI 每盘线程、读取块、每盘队列配置和任务快照。
- 逐盘状态与吞吐。

### M4：全文件 SHA-512 调度

- 从 RocksDB 领取全部新/变化文件。
- 增量复用、硬链接共享、文件变化重试。

### M5：计算线程

- GUI 全局计算线程配置和计算令牌。
- GUI FFmpeg 单任务线程配置和内部线程限制。
- 图片、视频和拼图任务公平运行。

### M6：十盘基准

- 单盘基准、混合盘并行、总线/CPU/RocksDB 瓶颈定位。
- 固化允许的线程范围和默认缓冲区。

### M7：HDD 顺序与坏块

- 实现 NTFS extent、卷物理区间和有界电梯排序。
- 实现 overlapped 读取、无进展超时、有限重试和 64 KiB 复试。
- 接入配置页面的排序窗口、重试次数、小块尺寸和超时快照。
- 实现不可读偏移日志、跳过文件和取消等待状态。
- 验证坏块错误不会生成部分 SHA-512，也不会自动暂停整盘。

## 12. 验收标准

1. 同一物理盘的多个分区共享读取通道。
2. 约 10 块在线独立磁盘可同时出现读取吞吐。
3. 每盘读取线程、全局计算线程、FFmpeg 单任务线程、读取块和队列容量按任务快照生效。
4. 所有新/变化文件进入完整 SHA-512，不做大小或快速指纹跳过。
5. 默认单 HDD 不因并发寻道显著低于单线程基线。
6. 总缓冲区和队列内存受配置上限约束。
7. 一个磁盘离线不停止其他磁盘。
8. 暂停、取消和恢复无死锁、不丢 RocksDB 检查点。
9. MySQL 延迟或断线不阻塞磁盘读取热路径。
10. GUI 能显示每盘线程、队列、当前文件、吞吐和错误。
11. NTFS HDD 能按有界窗口物理区间排序；非 NTFS 自动退化。
12. 坏块或读超时文件写日志、标记失败并跳过，后续文件继续。
13. 任一读取缺失都不会产生 SHA-512。
14. 千万级任务不会为 HDD 排序一次性载入全部文件。
15. HDD 排序窗口、坏块重试、小块尺寸和无进展超时均由配置页面传入且恢复时保持原值。

## 13. 风险与对策

| 风险 | 对策 |
| --- | --- |
| 10 块盘同时读导致 CPU 饱和 | 全局计算线程和 FFmpeg 令牌 |
| HDD 线程配置过高 | 默认 1、性能警告、配置范围和基准 |
| RocksDB 写放大 | 批量写、键设计、压缩和缓存基准 |
| MySQL 阻塞扫描 | RocksDB 持久化待同步、异步批量写 |
| 拓扑识别不准 | 明确 Unknown/Virtual，用户覆盖但不虚构物理关系 |
| 坏道导致工作线程长时间阻塞 | overlapped I/O、无进展超时和 `CancelIoEx`，不强杀线程 |
| 坏块重试进一步损耗机械盘 | 默认仅两级有限重试，失败立即跳过当前文件 |
| LCN 查询开销抵消收益 | 只对 HDD、按有界窗口查询并用真实基准决定窗口大小 |
