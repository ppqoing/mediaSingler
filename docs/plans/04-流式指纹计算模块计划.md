# 流式指纹与媒体数据模块计划

## 1. 模块目标

为所有文件提供内存有界、支持大文件、取消和进度的完整 SHA-512；为图片提供 dHash；为视频提供六帧 dHash、媒体元数据、静态画面判断和单个 2×3 拼图缩略图。模块不提取音轨数据，也不进行音频声纹去重。

## 2. 当前能力与必须修正项

可复用内容：FFmpeg 集成、图片 dHash、截图均匀采样、BCrypt SHA-512 和 `__popcnt` 汉明距离。

必须修正：

1. `ComputeFileSHA512` 当前整文件读入 `std::vector<uint8_t>`。
2. 单次 `BCryptHashData` 长度为 `ULONG`，需要固定块循环。
3. API 缺少取消、进度、文件变化和详细错误。
4. 现有截图函数生成多个文件，不符合单个 2×3 拼图。
5. 现有 dHash 注释中存在 17×16 描述，实际算法是 9×8，需要统一。
6. 媒体结果没有算法版本和 RocksDB/MySQL 同步状态。

## 3. 全文件完整 SHA-512

### 3.1 算法规范

- 算法 ID：`full-sha512-v1`。
- 对每个新文件或内容版本读取全部字节。
- 使用固定块增量调用 BCrypt。
- 摘要以 64 字节二进制存储，UI/导出时转为 128 位十六进制字符。
- 零字节文件也计算标准 SHA-512。
- 不使用大小筛选或快速采样代替完整哈希。

### 3.2 文件稳定性

读取前后检查卷、文件 ID、大小和最后修改时间。任一变化时返回 `FileChangedDuringRead`，不提交有效 SHA-512。新路径即使文件 ID 或名称相似也执行 SHA-512；命中已有摘要后再复用内容数据。

### 3.3 I/O

- 使用 `CreateFileW` 和 Unicode 长路径。
- 完整哈希使用 `FILE_FLAG_SEQUENTIAL_SCAN`；HDD 调度顺序由上层物理区间规划。
- 使用 overlapped 块读取以支持无进展超时和 `CancelIoEx`。
- 默认共享读/写/删除，并依赖前后稳定性检查。
- 缓冲区来自调度层固定池，建议基准 1 至 8 MiB。
- 每块后检查暂停和取消。

### 3.4 坏块和超时语义

- 坏块类 Win32 错误由调度策略决定有限重试；哈希核心只接受完整、有序、无缺口的字节流。
- 原块重试失败后允许以 64 KiB 小块定位和复试，但任何小块最终失败都会使整个文件摘要无效。
- 默认 60 秒无成功读取进展后请求取消当前 overlapped I/O；超时值由配置传入。
- 返回结果包含失败偏移、请求长度、成功读取字节、系统错误和是否超时。
- 不用零填充、跳过坏区或拼接剩余内容生成伪 SHA-512。
- 上层把 `Unreadable/ReadTimeout` 和坏块信息写文件日志后继续下一文件。

### 3.5 兼容 API

保留 `ComputeFileSHA512` C ABI，内部改为流式核心。`DedupCore` 使用类型化结果，包含状态、系统码、读取字节、耗时和源文件快照。

## 4. 图片 dHash

### 4.1 规格

- 算法 ID：`image-dhash64-v1`。
- 原图按现有 FFmpeg 路径解码并缩放为 9×8 灰度。
- 每行比较 8 对相邻像素，共 64 位。
- 不应用 EXIF 旋转归一化。
- 内部保存 `uint64_t`，展示时输出 16 位十六进制。

### 4.2 判定

两张图片的汉明距离严格 `< 5` 时形成内容相似候选。dHash 只在用户手动触发内容去重时参与匹配，但首次创建 `sha512_file_data` 时可提前计算和缓存。

## 5. 视频六帧指纹

### 5.1 采样

- 算法 ID：`video-dhash6-v1`。
- 时长为 `t`，采样点为 `t/7、2t/7、3t/7、4t/7、5t/7、6t/7`。
- 每个采样点从前一个关键帧 seek 后解码到目标附近。
- 记录目标时间和实际帧时间，失败点不伪造摘要。
- 每帧直接缩放为 9×8 灰度计算 dHash。

### 5.2 视频相似判定

两个视频必须满足：

1. 都不是静态画面视频。
2. 时长绝对差不超过 2 秒。
3. 六个对应采样帧都有有效 dHash。
4. 六个对应汉明距离的算术平均值严格 `< 5`。

视频分辨率、码率和文件大小用于保留质量排序，不改变上述相似判定。

### 5.3 静态画面判定

六个帧 dHash 形成 15 个两两组合。15 个汉明距离的算术平均值严格 `< 5` 时，内容记录标记 `StaticVisual=true`。此类视频不进入视频 dHash 相似匹配，只参与 SHA-512 精确去重。

## 6. 2×3 拼图缩略图

### 6.1 生成规范

- 算法 ID：`video-contact-sheet-2x3-v1`。
- 使用与 dHash 相同的六个采样帧。
- 按时间顺序从左到右、从上到下排列为 2 行 × 3 列。
- 所有格子使用统一画布尺寸和保持宽高比的缩放；空白区域使用固定背景色。
- 一个视频只生成一个拼图文件，不保存六张独立缩略图。
- 缩略图根目录、格式和每格长边在配置页面设置；默认 JPEG、256 像素。
- 文件名由 SHA-512 和拼图算法版本派生，避免移动路径造成重复缩略图。

### 6.2 写入和复用

- 先写同目录临时文件，再原子改名，避免半张拼图。
- MySQL `sha512_file_data` 保存拼图路径和规格版本，规格版本包含格式、单格尺寸和布局。
- 新路径命中已有 SHA-512 时复用拼图。
- 配置页面修改缩略图尺寸后生成新的规格版本；拼图文件缺失或版本变化时按需重建，不重新计算 SHA-512、dHash 或媒体元数据。

## 7. 媒体元数据

### 7.1 图片

- 宽、高、检测格式、MIME、dHash。

### 7.2 视频

- 容器格式、时长、宽、高、帧率、视频编码、视频码率、像素格式。
- 六个 dHash、实际采样时间、静态画面标记、拼图路径和算法版本。

### 7.3 不提取

- 音频编码、声道、采样率、音频码率和声纹。
- 图片/视频旋转归一化结果。

音频和其他文件仍保存 SHA-512、大小、扩展名和基础类型。

## 8. 线程与资源

- 每个 FFmpeg 任务独占 `AVFormatContext`、`AVCodecContext`、`SwsContext`、帧和包。
- 上下文不跨线程共享。
- 任务开始前取得物理盘读取令牌和全局计算令牌。
- FFmpeg 单任务内部解码线程数从配置页面进入任务快照，并受全局计算线程预算约束。
- 六个采样帧逐个处理；完成 dHash 和拼图格写入后释放帧内存。
- 拼图编码结束后只保留文件路径和小型元数据。
- GUI 纹理加载不属于本模块。

## 9. RocksDB 和 MySQL 数据流

1. SHA-512 和媒体结果先以版本化值写 RocksDB。
2. 同一写批次创建或更新 MySQL 待同步消息。
3. MySQL `file_path_sha512` 保存路径映射。
4. MySQL `sha512_file_data` 以 SHA-512 为主键幂等插入内容数据。
5. 已存在且算法版本有效时不重新解码。
6. 媒体失败不影响 SHA-512 路径映射；保存失败状态供重试。

## 11. 实施任务

### M1：流式 SHA-512

- 替换整文件读取。
- 支持大文件、overlapped I/O、取消、无进展超时、坏块错误明细、进度和稳定性检查。
- 保持旧 C ABI。

### M2：图片 dHash 统一

- 统一 9×8 规格、位序、测试向量和无旋转规则。

### M3：视频六帧

- 固化时间点、实际时间记录、六个 dHash 和静态判定。
- 接入配置页面的 FFmpeg 单任务线程数，受全局计算预算限制并覆盖失败路径。

### M4：2×3 拼图

- 六帧复用、统一格子、原子文件写入和 SHA-512 命名。
- 接入配置页面的缩略图目录、格式和单格尺寸，保存规格版本并支持按需重建。

### M5：媒体元数据

- 图片和视频字段。
- 明确排除音轨。
- RocksDB 版本化与 MySQL 幂等同步。

## 12. 验收标准

1. 所有新/变化文件完整计算 SHA-512。
2. 大于 4 GiB 文件内存不随文件大小增长，结果与权威工具一致。
3. 读取中变化的文件不提交有效摘要。
4. 图片 dHash 不做旋转归一，汉明距离实现与测试向量一致。
5. 每个视频固定生成六个 dHash和一个 2×3 拼图文件。
6. 视频相似只在时长差不超过 2 秒且六组平均距离 `< 5` 时成立。
7. 静态画面视频按 15 组平均距离 `< 5` 标记，并跳过视频相似匹配。
8. 不提取或保存音轨数据。
9. 新路径命中已有 SHA-512 时复用媒体数据和拼图。
10. 取消和错误路径正确释放 FFmpeg、BCrypt、文件和缓冲区资源。
11. 坏块或超时不会产出部分 SHA-512，结果包含准确失败偏移和系统错误。
12. 跳过坏块文件后，调度器仍能继续处理同一机械盘的后续文件。
13. 配置的 FFmpeg 线程数和视频拼图单格尺寸在新任务中准确生效。
14. 缩略图规格变化只重建拼图，不重算 SHA-512、dHash 或媒体元数据。

## 13. 风险与对策

| 风险 | 对策 |
| --- | --- |
| 全文件 SHA-512 I/O 量大 | 多物理盘并行、盘内配置线程、增量复用 |
| 视频 seek 帧偏移 | 保存实际时间，固定版本和测试样本 |
| 静态水印影响帧距离 | 使用 15 组平均距离 `< 5`，保留阈值版本 |
| 拼图占用内存 | 逐格处理、统一缓冲、完成后立即释放 |
| 缩略图目录失效 | 独立重建任务，不破坏内容身份 |
| MySQL 断线 | 先写 RocksDB，待同步持久化重试 |
| 坏块产生错误摘要 | 任一字节读取失败即作废整个 SHA-512，不允许填充或跳字节 |
| 驱动不响应取消 | 保持 `CancelPending`，不强杀线程，其他磁盘继续 |
