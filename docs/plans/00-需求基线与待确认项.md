# 已确认需求基线

## 1. 扫描范围

1. GUI 支持添加、浏览、排序和移除多个扫描路径。
2. 扫描路径在任务开始时冻结；运行中修改只影响下一次扫描。
3. 从配置中移除路径后，历史记录保留并标记为不在当前范围。
4. 目标环境约 10 块物理磁盘，包含 HDD 和 SSD。
5. 每个物理磁盘可单独配置文件读取线程数。
6. 可配置全局计算线程数。
7. 扫描支持暂停、继续、取消、断点续扫和增量扫描。
8. 单次任务可能包含千万级文件路径，SHA-512、图片 dHash、视频 dHash 均不得全量两两比较。

## 2. SHA-512 精确去重

1. 所有发现的文件都计算完整 SHA-512，不限媒体类型，也不因文件大小唯一而跳过。
2. 音频、图片、视频和其他文件都参与 SHA-512 精确去重。
3. MySQL 自动按 SHA-512 读取并形成精确重复组。
4. 同一 SHA-512 的媒体信息只保存一份；不同路径通过独立映射表关联。
5. 文件移动或出现新路径时仍计算 SHA-512；若 MySQL 已存在该 SHA-512，则只新增路径映射并复用媒体数据，不重新计算 dHash 和媒体元数据。

## 3. 图片内容去重

1. 图片解码后计算 64 位 dHash。
2. 不进行 EXIF 旋转归一化。
3. 两张图片的汉明距离严格 `< 5` 时形成内容相似候选。
4. 内容相似扫描由用户在 GUI 手动触发。
5. dHash 相似不等同 SHA-512 精确相同，但允许进入人工审阅和批量删除流程。

## 4. 视频内容去重

1. 每个视频在 `t/7、2t/7、3t/7、4t/7、5t/7、6t/7` 六个等间隔时间点取帧。
2. 每帧计算一个 64 位 dHash。
3. 六帧合成为一个 2 行 × 3 列的拼图缩略图文件；数据库只保存拼图路径。
4. 两个视频时长差超过 2 秒时不比较 dHash。
5. 时长差不超过 2 秒时，比较六组对应帧；六个汉明距离的平均值严格 `< 5` 时形成视频内容相似候选。
6. 单个视频六帧全部 15 组两两汉明距离的平均值 `< 5` 时，标记为静态画面视频。
7. 静态画面视频不参与视频 dHash 内容相似去重，只参与 SHA-512 精确去重。
8. 视频不进行画面旋转归一化。

## 5. 音频规则

1. 音频文件计算完整 SHA-512并参与精确去重。
2. 不提取音轨媒体数据。
3. 不计算音频声纹，不进行音频内容相似去重。

## 6. 媒体服务器数据

### 6.1 路径级数据

- 原始路径、规范化路径、SHA-512。
- 扫描根目录、物理磁盘、卷和文件 ID。
- 文件大小、扩展名、创建时间、修改时间。
- 在线、缺失、不在范围和同步状态。
- 首次发现、最后发现和所属扫描任务。

### 6.2 SHA-512 内容级数据

- SHA-512、内容大小、媒体类型、MIME 和容器格式。
- 图片：宽、高、格式和 dHash。
- 视频：时长、宽、高、帧率、视频编码、视频码率、像素格式、六个 dHash、静态画面标记和 2×3 拼图路径。
- 不保存音轨编码、声道、采样率或音频码率。

缩略图保存为文件。配置页面可设置缩略图根目录、格式和每格尺寸；默认 JPEG、每格长边 256 像素。

## 7. 数据库与同步

1. 使用一个持久化 RocksDB 保存本地扫描任务、队列状态、指纹结果、MySQL 待同步数据和断点；通过 `scan_id` 区分扫描任务。
2. 使用 MySQL 8.0+ 保存共享路径映射、SHA-512 内容数据、重复组和配置版本。
3. 扫描优先写 RocksDB，再按批次异步同步 MySQL。
4. MySQL 不可用时扫描继续；未同步数据保留在 RocksDB，连接恢复后重试。
5. GUI 必须区分本地完成和 MySQL 同步完成。
6. 配置页面支持 MySQL 主机、端口、数据库名、用户名、密码、TLS、连接池、超时和 `mysqldump` 路径。
7. 配置页面只提供“初始化数据库表”按钮，执行 `CREATE TABLE IF NOT EXISTS`；不提供一键删除或重建表按钮。
8. 如初始化前需要备份既有应用表，可调用 `mysqldump`，备份目录可配置；备份失败不得破坏现有表。

## 8. MySQL 核心表边界

1. `file_path_sha512`：一条路径对应一个 SHA-512，保存路径级信息和状态。
2. `sha512_file_data`：一条 SHA-512 对应一份共享内容和媒体数据，只增量写入，不因路径删除而删除。
3. 重复组和成员使用独立表，不把 UI 临时勾选写入内容表。
4. 删除本地文件成功后，在整个删除批次末尾直接删除对应 `file_path_sha512` 行。
5. 删除失败的路径映射保留。
6. 删除、移动、数据库初始化和同步动作写滚动文件日志，不建立数据库删除审计表。

## 9. 删除与保留

1. 支持 SHA-512 精确重复组和 dHash 相似组的永久删除。
2. 选择条件按钮与“删除选中文件”按钮分离，条件按钮只勾选，不立即删除。
3. 支持按文件体积选择较大版本。
4. 支持按质量选择较低版本：视频按分辨率像素数、视频码率、文件大小排序；图片按分辨率像素数、文件大小排序。
5. 支持按指定物理磁盘批量选择。
6. 支持按最新、最旧、最小、最大和路径优先级选择保留文件；最新/最旧使用最后修改时间。
7. GUI 中扫描路径可拖动排序，顺序用于路径保留优先级。
8. 每个重复组在执行前必须保证至少一个成员未被删除；若规则选中全部成员，系统自动撤销最高保留优先级成员的选中状态。
9. 永久删除前展示预览、组类型、保留成员、待删除路径和预计释放空间，并进行二次确认。
10. dHash 相似组在 UI 展示图片缩略图或视频 2×3 拼图。
11. 本地永久删除成功后记录日志并进入映射删除批次；日志不可写时禁止开始删除批次。

## 10. 增量与状态

1. 文件未变化且 RocksDB/MySQL 数据有效时复用已有结果。
2. 新路径仍计算 SHA-512；命中已有内容记录后只新增路径映射。
3. 文件缺失、磁盘离线或移出扫描范围时保留本地状态标记，不立即删除共享内容数据。
4. 本地文件被程序永久删除后，路径映射在删除批次末尾从 MySQL 直接删除。
5. 路径映射删除失败时在 RocksDB 保留待重试操作，避免下次扫描恢复已删除路径。

## 11. 机械硬盘坏块与读取顺序

1. HDD 默认使用一个读取线程、顺序大块读取和 `FILE_FLAG_SEQUENTIAL_SCAN`。
2. NTFS 文件允许通过 `FSCTL_GET_RETRIEVAL_POINTERS` 获取逻辑簇位置，并结合卷磁盘区间换算物理起始位置。
3. HDD 在有界排序窗口内使用电梯算法按物理位置读取；不把千万级文件全部放入内存排序。
4. 非 NTFS、物理区间查询失败、虚拟盘或复杂卷退化为稳定路径顺序。
5. SSD 不执行 LCN/物理区间排序。
6. 遇到 `ERROR_CRC`、`ERROR_IO_DEVICE`、`ERROR_SECTOR_NOT_FOUND` 等错误时，当前块按配置有限重试，再缩小到 64 KiB 重试。
7. 重试仍失败时，当前文件标记 `Unreadable`，不产生部分 SHA-512，并继续下一个文件。
8. 单次异步读取连续 60 秒没有进展时，默认调用 `CancelIoEx` 尝试取消并跳过文件；超时值可配置。
9. 不因连续坏块文件熔断或自动暂停整个物理盘。
10. 坏块文件日志记录路径、物理盘、读取偏移、长度、Win32 错误、重试次数和超时状态。

## 12. 千万级哈希比较

1. SHA-512 精确重复使用二进制摘要索引和按摘要有序的单遍聚合，复杂度接近 O(N)，不做路径两两比较。
2. 图片 dHash 阈值 `< 5` 等价于汉明距离 0 至 4；把 64 位哈希拆成 6 段并建立全部 15 个两段联合键。最多四段被差异位污染，因此至少有两段完全一致，候选召回后再进行真实距离复核。
3. 视频先按时长差不超过 2 秒筛选。平均距离 `< 5` 等价于六帧距离和不超过 29，因此至少一组对应帧距离不超过 4；对六个帧位分别使用 15 个两段联合键，共 90 个候选键，再做真实六帧复核。
4. 上述二级索引持久化在 RocksDB，按 SHA-512 唯一内容而不是每条路径建立，减少精确副本造成的重复索引。
5. 索引构建、候选对去重和结果生成均使用键集分页、有界批次和断点，不能一次加载千万级记录。

## 13. 配置页面

1. 运行参数必须集中显示在配置页面，不以隐藏常量作为唯一修改方式。
2. 所有配置页面字段保存到程序安装目录下的 `config.json`，文件使用 UTF-8 JSON，并包含 `schema_version`。
3. MySQL 密码使用当前 Windows 用户范围的 DPAPI 加密，将密文 Base64 后写入 JSON；JSON、日志和错误信息中不得出现明文密码。
4. 保存时先完成整份配置校验，再写同目录临时文件、刷盘并原子替换 `config.json`；保留上一份有效的 `config.json.bak`，不能直接截断覆盖有效配置。
5. 配置文件不存在时加载内置默认值；主文件损坏时报告错误并尝试只读加载备份；主文件和备份都无效时加载未保存的默认值，未经用户确认不得覆盖损坏文件。
6. 安装目录不可写、磁盘空间不足或原子替换失败时，保存操作失败并保留原有效文件；不得静默回退到其他目录。
7. `config.json` 保存配置页面值，不保存扫描进度、工作队列、指纹结果或待同步操作；这些运行状态仍由 RocksDB 持久化。
8. 支持已知旧模式向当前模式迁移；遇到高于程序支持范围的 `schema_version` 时只读报错，不得用旧程序覆盖。
9. 线程配置包括每个物理盘的读取线程数、全局计算线程数和 FFmpeg 单任务内部线程数。
10. 读取与调度配置包括读取块大小、每盘队列容量、HDD 物理区间优化开关和 HDD 排序窗口大小。
11. 坏块配置包括原块重试次数、小块重试次数、小块尺寸和无读取进展超时；默认原块重试 1 次、小块重试 1 次、小块为 64 KiB、无进展超时为 60 秒。
12. MySQL 配置除连接信息外，还包括连接池大小、连接超时、命令超时、重试间隔和同步批量大小。
13. 缩略图配置包括输出目录、格式、视频 2×3 拼图单格长边、图片结果预览长边、缓存条目数、内存上限和显存上限；视频单格长边默认 256 像素。
14. GUI 必须显示数值单位、合法范围、预计线程总数和预计内存预算；非法配置不得启动任务。
15. 任务开始时冻结完整配置快照并写入 RocksDB；运行中修改只影响下一任务，恢复任务继续使用原快照。
16. 缩略图尺寸变化提升缩略图规格版本，已有拼图按需重新生成，不重新计算 SHA-512 或 dHash。
17. dHash `< 5`、视频六帧采样点、时长差不超过 2 秒和静态画面规则是固定算法规则，不在配置页面开放修改。

## 14. 不包含的能力

- 不进行音频声纹去重。
- 不保存或分析音轨信息。
- 不对图片或视频做旋转归一化。
- 不提供 MySQL 一键重建或清空表。
- 不把六张视频缩略图保存为六个独立图片。
- 不把删除审计写入数据库。
- 不允许一个删除批次删除重复组的全部成员。
- 不尝试修复或恢复坏道中的数据，也不允许跳过坏字节后生成 SHA-512。
- 不对千万级 SHA-512 或 dHash 执行 O(N²) 全量两两比较。
