# 测试、性能与交付计划

## 1. 目标

证明工具满足以下范围：所有文件完整 SHA-512；约 10 块 HDD/SSD 并行；HDD 物理顺序优化和坏块跳过；千万级 SHA-512、图片 dHash、视频 dHash 索引；RocksDB 断点与 MySQL 8.0+ 补同步；视频六帧和 2×3 拼图；永久删除且每组保留一个。

当前项目是 Visual Studio C++ 项目，不是 Unity 项目。实现和验证使用 MSBuild/Visual Studio 工具链；当前 Release 构建与自动化测试已执行，真实 MySQL 和多盘环境仍按本计划验收。

## 2. 构建范围

- 主交付：`Release|x64`。
- 诊断：`Debug|x64` 和必要的 AddressSanitizer 配置。
- x86 不作为交付目标，保持当前解决方案和依赖统一为 Windows x64。
- 固定并记录 FFmpeg、RocksDB、MySQL 客户端版本；Everything SDK 只属于诊断工具。
- Release 保留可用 PDB 和崩溃诊断信息。
- 新文件纳入 `.vcxproj` 和 filters，不写本地绝对依赖路径。

## 3. 单元测试

### 3.1 路径与发现

- 普通、根目录、重叠、中文、emoji、长路径和不同大小写。
- reparse point、目录联接循环和根目录越界。
- 同一文件硬链接和路径移动。
- `Missing`、`Offline`、`OutOfScope` 和 `ProgramDeleted`。
- 新路径必须进入 SHA-512；未变化内容可以增量复用。

### 3.2 存储拓扑和 HDD 顺序

- 同一物理盘不同分区共享 `StorageTargetKey`。
- HDD、SSD、Unknown 和复杂卷退化。
- 模拟 `FSCTL_GET_RETRIEVAL_POINTERS` 返回单 extent、多 extent、稀疏和失败。
- 用卷起始区间和每簇字节换算物理位置。
- 有界窗口电梯顺序、方向反转、键集游标和跨窗口恢复。
- SSD 不调用 LCN 排序路径。

### 3.3 坏块和超时

通过可注入文件读取适配器模拟：

- `ERROR_CRC`、`ERROR_IO_DEVICE`、`ERROR_SECTOR_NOT_FOUND`。
- 原块重试成功。
- 原块失败、64 KiB 小块重试成功。
- 最终失败返回准确偏移和长度。
- 60 秒无进展触发 `CancelIoEx`；单元测试使用缩短虚拟时钟。
- 驱动延迟取消进入 `CancelPending`，不调用 `TerminateThread`。
- 失败文件没有 SHA-512，后续文件继续。
- 连续多个失败不会触发整盘熔断或自动暂停。
- 坏块文件日志字段完整。

### 3.4 SHA-512

- 空文件、小文件、恰好一块、多块和稀疏大于 4 GiB 文件。
- 流式结果与权威 SHA-512 一致。
- 文件读取中修改、截断、替换和权限失败。
- 暂停、取消、坏块、超时均不提交部分摘要。
- 二进制摘要和十六进制展示转换。

### 3.5 图片 dHash

- 9×8 灰度规格、位序和固定测试向量。
- 明确不进行 EXIF 旋转归一化。
- 距离 0 至 4命中，距离 5 不命中。
- 6 段 `11/11/11/11/10/10` 的 15 个两段联合键保证所有距离不超过 4 的样本至少共享一个联合桶。
- 热门完整 dHash 聚合不生成组合级候选集合。

### 3.6 视频

- 采样点严格为 `t/7` 至 `6t/7`。
- 六个实际时间、六个 dHash 和算法版本。
- 时长差 2 秒边界参与比较，超过边界不比较。
- 六帧距离和 29 命中，距离和 30 不命中。
- 15 组单视频距离平均 `< 5` 标记静态画面。
- 静态画面只参与 SHA-512。
- 只生成一个 2 行 × 3 列拼图，帧顺序正确。
- 不提取音轨数据。

### 3.7 千万级索引算法

- SHA-512 有序单遍聚合与增量受影响摘要。
- 图片两段联合键、完整视觉签名压缩和候选复核。
- 视频时长桶、六帧三十段 posting 和候选复核。
- 候选对规范化、重复桶去重和断点。
- 算法版本切换后新旧索引隔离。
- 新路径复用旧 SHA-512 时不重复建内容索引。

### 3.8 选择和永久删除

- 较大、低质量、指定磁盘、最新、最旧、最小、最大和路径优先级。
- SHA-512 精确组禁用“较大”条件。
- 任意条件组合后每组至少保留一个。
- 本地删除失败不删除路径映射。
- 本地删除成功在批次末尾删除映射。
- MySQL 删除失败生成 RocksDB tombstone 并可重试。
- `sha512_file_data` 永不级联删除。
- 日志不可写时删除批次拒绝开始。

### 3.9 状态机

- 扫描、内容匹配和删除批次所有合法/非法转换。
- 重复、迟到和错误任务 ID 命令。
- 暂停、取消、崩溃恢复和幂等重放。

### 3.10 配置模型

- 安装目录没有 `config.json` 时加载内置默认值，首次显式保存后重启可完整往返。
- UTF-8 JSON 序列化、字段单位、路径中的中文/空格/反斜杠和 `schema_version`。
- 已知旧模式迁移；未知高版本只读拒绝且不覆盖原文件。
- 临时文件写入、flush、原子替换和 `config.json.bak` 保留；在每个失败点验证旧主文件仍有效。
- 主文件截断、非法 JSON、类型错误和越界值时告警并尝试备份；主文件与备份均损坏时不自动覆盖。
- 安装目录只读、磁盘满、文件被占用和替换失败时返回准确错误，不回退到其他路径。
- DPAPI `CurrentUser` 加解密往返、Base64 非法值和解密身份不匹配；扫描 JSON、RocksDB 和日志确认不存在测试明文密码。
- 每物理盘读取线程、全局计算线程和 FFmpeg 单任务线程的最小值、最大值和非法值。
- 读取块、每盘队列、HDD 排序窗口、坏块重试、小块尺寸和无进展超时的单位与边界。
- MySQL 连接/命令超时、重试间隔、同步批量和连接池的组合校验。
- 视频拼图单格尺寸、图片预览尺寸、缓存条目、内存与显存预算的组合校验。
- 预计线程数和内存预算计算与实际分配一致。
- 任务开始后配置快照不可变，页面修改只影响新任务；崩溃恢复使用原快照。
- 缩略图尺寸或格式改变只提升规格版本并重建缩略图，不重新计算 SHA-512 或 dHash。
- dHash 阈值、视频六帧采样和 2 秒时长规则没有普通可编辑控件。

## 4. RocksDB 测试

- Column Family 创建、打开、版本迁移和未知高版本拒绝。
- `WriteBatch` 原子更新文件结果、工作项、进度和同步消息。
- WAL 损坏、磁盘满、只读目录和写失败。
- 进程强制退出后恢复扫描、索引和 tombstone。
- compaction、posting list 分片和过期消息清理。
- 单个持久库中的多 `scan_id` 隔离。
- 千万级生成数据下的数据库大小、写放大和恢复时间。

## 5. MySQL 测试

### 5.1 模式

- MySQL 8.0+ 新库初始化。
- 已存在表的幂等初始化。
- 向前迁移前 `mysqldump` 成功/失败。
- 配置页没有清空或重建入口。
- `file_path_sha512` 与 `sha512_file_data` 独立。
- 长路径摘要冲突时比较完整路径。

### 5.2 同步

- MySQL 在线批量 upsert。
- 断线期间扫描继续写 RocksDB。
- 恢复后顺序、幂等补同步。
- 重复重放不增加重复路径或内容行。
- 内容先于路径同步。
- 映射按 `path_id + expected_sha512` 删除。
- 本地完成与远端完成状态分离。

### 5.3 千万级查询

- `file_path_sha512(sha512, path_id)` 覆盖索引执行计划。
- SHA-512 顺序键集扫描，不使用大 OFFSET。
- 内容表主键键集分页流入 RocksDB 索引。
- 精确组成员查询和结果分页。
- MySQL 网络延迟不会阻塞文件读取线程。

## 6. 集成测试

### 6.1 首次扫描闭环

“路径配置 → Win32 流式发现 → HDD/SSD 调度 → 全文件 SHA-512 → RocksDB → MySQL → 精确组 → 媒体数据 → 手动 dHash → 报告”。数据集包含精确副本、同大小不同内容、相似图片、转码视频、静态视频、音频、损坏媒体和非媒体文件。

### 6.2 增量与移动

- 无变化重扫只复用。
- 文件变化重新 SHA-512。
- 新路径/移动文件重新 SHA-512，命中内容表后复用 dHash 和拼图。
- 缩略图文件缺失只重建拼图。
- 盘离线不删除映射。

### 6.3 删除闭环

只在受控临时目录执行：

- 批量条件选择和手工调整。
- 图片/视频相似组显示缩略图。
- 二次确认、永久删除和日志。
- 每组保留一个。
- 部分本地失败、MySQL 失败和重启补删。
- 映射删除后内容表保留。

### 6.4 十盘故障

- 约 10 个独立存储目标同时工作。
- 一个 HDD 注入坏块错误，后续文件和其他盘继续。
- 一个 worker `CancelPending`，其他通道继续。
- MySQL 断线、RocksDB 正常。
- RocksDB 失败时全部生产者安全收敛。

## 7. 性能基准

### 7.1 硬件矩阵

- 单 HDD、单 SATA SSD、单 NVMe。
- 双 HDD、HDD+SSD、双 SSD。
- 接近用户环境的约 10 块混合磁盘。
- 如存在 USB、虚拟盘或 Storage Spaces，单独记录。

### 7.2 数据集

- 少量超大视频。
- 大量小文件。
- 千万级合成路径与摘要记录。
- 高 SHA-512 重复率。
- 图片热门 dHash 大桶。
- 视频相近时长密集桶。
- NTFS 碎片/多 extent 样本。
- 模拟坏块和超时读取。

### 7.3 指标

- 发现文件/秒、每盘 MB/s、总读取 MB/s。
- HDD 寻道次数近似指标、LCN 命中率和排序窗口收益。
- 坏块重试时间、超时取消时间和跳过文件数。
- SHA-512 文件/秒与字节/秒。
- RocksDB 写入放大、compaction、索引大小和恢复时间。
- MySQL 批量吞吐、待同步峰值和补同步速度。
- 千万级精确聚合耗时和峰值内存。
- 图片/视频索引构建速度、候选缩减率和真实距离计算数。
- GUI 帧时间、纹理缓存字节和淘汰数量。
- 永久删除文件/秒和映射批量删除耗时。

## 8. 性能门槛

在首次真实基准前不编造固定 MB/s，但必须满足结构性门槛：

1. 多个独立磁盘在重叠时间内同时产生吞吐，优于串行总耗时。
2. 单 HDD 默认物理顺序优化不低于稳定路径顺序基线；若 LCN 查询收益为负则可自动退化。
3. 所有文件 SHA-512 的内存与文件大小无关。
4. 千万级 SHA-512 分组只进行 O(N) 级有序扫描，不出现 N² 计数器。
5. 图片/视频只对索引候选计算真实距离，并用穷举小样本证明无漏候选。
6. RocksDB/MySQL 数据规模扩大十倍时，进程内存由有界队列控制，不随记录总数线性全量驻留。
7. MySQL 离线不降低本地读取线程到等待网络状态。
8. GUI 缩略图内存/显存不超过配置预算。
9. 用配置页面改变线程、块、队列和同步批量后，实际运行指标使用新任务快照中的值，并始终保持有界资源。

## 9. 正确性与安全验收门

- 每个新内容版本都有完整 SHA-512 或明确 `Unreadable/ReadTimeout`，没有部分摘要。
- 图片、视频和静态视频阈值边界正确。
- 音频没有声纹或音轨数据。
- 每个视频只有一个 2×3 拼图。
- 每组永久删除后至少保留一个文件。
- 删除失败不删映射，内容表不级联删除。
- 删除和坏块进入文件日志。
- 日志不可写时不执行永久删除。
- 配置非法时任务不能启动；恢复任务不会被当前页面配置改写。
- 用户设置的缩略图尺寸在新规格拼图和图片预览中准确生效。
- JSON 保存中断或失败不破坏上一有效配置，MySQL 密码不以明文落盘。

## 11. 阶段完成标准

每个路线阶段只有在对应代码、测试、错误路径、性能证据和文档一致时才完成。窄范围单元测试不能代替十盘、千万级、MySQL 离线、坏块或永久删除的范围性证据。
