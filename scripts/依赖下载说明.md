# 依赖手动下载说明

## third_party/ffmpeg 运行库

仓库只保留 third_party/ffmpeg 的头文件（include）与导入库（lib）；体积较大的 DLL 与
可执行文件（bin 目录）不入库。克隆后执行一次即可还原：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-ThirdParty.ps1
```

脚本下载 BtbN FFmpeg-Builds 的 win64 GPL shared 压缩包并只还原 bin 目录；完成后会校验
仓库 include/lib 版本锁定的 DLL 集合（avcodec-63 等），主版本漂移时中止并提示。需要代理：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-ThirdParty.ps1 -Proxy http://127.0.0.1:7890
```

## vcpkg 依赖

默认只下载 `vcpkg.json` 固定基线中的全部直接和传递依赖，不进行编译：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-Dependencies.ps1
```

只检查脚本参数并显示将执行的命令，不联系网络：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-Dependencies.ps1 -DryRun
```

如果已经安装 aria2，建议启用多连接和断点续传：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-Dependencies.ps1 `
  -Aria2Path D:\Tools\aria2\aria2c.exe `
  -Connections 16
```

需要代理时显式传入，不会把代理地址写进项目配置：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-Dependencies.ps1 `
  -Proxy http://127.0.0.1:7890
```

下载完成后，添加 `-Install` 执行编译和安装：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Download-Dependencies.ps1 -Install
```

默认目录：

- vcpkg 可执行文件：`E:\vcpkg\vcpkg.exe`
- 下载缓存：项目根目录 `.downloads\vcpkg`
- 安装结果：`E:\vcpkg\installed`

脚本特点：

- 由 vcpkg 根据固定 `builtin-baseline` 解析版本，不手工维护容易漂移的 URL 清单。
- 每个文件先写入 `.part`，SHA-512 与端口清单一致后才进入正式缓存。
- 中断后可重复运行；curl 和 aria2 都支持续传。
- 默认不修改系统环境变量，不执行全局 `vcpkg integrate install`。
